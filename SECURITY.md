# Security Policy

## Supported versions

Security fixes are provided for the latest state of the `main` branch.

## Reporting a vulnerability

Do not open public issues for undisclosed vulnerabilities.

Instead:

1. Contact the maintainer privately through the repository security reporting channel if it is enabled.
2. If a private reporting channel is not available, use the maintainer contact details published on the repository profile.
3. Include affected version or commit, reproduction details, impact, and any required environment notes.

## Scope expectations

Reports are most helpful when they describe issues in:

- authorization or scope enforcement
- provider or credential handling
- report generation or persistence
- module execution safety boundaries
- TUI or logging paths that may expose sensitive data

Please avoid sending real secrets or third-party data in reports unless explicitly requested through a secure channel.
