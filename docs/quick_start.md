# Quick Start\n\nThis project currently runs as a CLI-driven autonomous agent (not an HTTP server entrypoint).\n\n## 1) Prerequisites\n\n- Go `1.26.2+`\n- Valid model provider credentials (OpenAI / Anthropic / Gemini), unless using a local compatible endpoint\n\n## 2) Build\n\nFrom repository root:\n\n```bash\ngo mod tidy\ngo build -o fire_starter ./cmd/fire_starter\n```\n\n## 3) Run\n\n```bash\n./fire_starter -target 192.168.1.10\n```\n\nWith explicit provider/model settings:\n\n```bash\n./fire_starter \\\n  -target http://127.0.0.1 \\\n  -provider openai \\\n  -model gpt-4o \\\n  -max-iters 15 \\\n  -verbose\n```\n\n## 4) Core runtime flow\n\n1. Decisions are loaded from `src/matrix/decisions.json` (`src/agent/workflow.go`).\n2. Tools are generated from decisions (`src/matrix/tool_registry.go`).\n3. Only tools relevant to the current phase are exposed to the model, plus control tools:\n   - `get_knowledge_graph`\n   - `advance_phase`\n   - `submit` (only when completion criteria are met)\n4. Tool outputs are fed back into the knowledge graph (`src/matrix/knowledge_graph.go`) to guide scoring and phase advancement.\n\n## 5) Important flags\n\n- `-target` (required): target IP/URL\n- `-provider`: `openai`, `anthropic`, `gemini`, `local`\n- `-model`: model ID\n- `-base-url`: custom provider endpoint\n- `-max-iters`: maximum control-loop iterations\n- `-verbose`: debug logging\n\n## 6) Test\n\n```bash\ngo test ./...\n```\n\nFor deeper checks while iterating:\n\n```bash\ngo test ./src/agent ./src/matrix ./src/modules\n```\n\n## 7) Common troubleshooting\n\n- **`go.mod requires go >= 1.26.2`**\n  - Upgrade local Go toolchain to 1.26.2 or newer.\n- **Model/provider initialization errors**\n  - Verify credentials and provider-specific environment setup.\n- **Tool execution returns target required errors**\n  - Ensure payload or target includes a valid `ip` or `url`.\n
